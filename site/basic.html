CSS:

  • STANDS FOR CASCADING STYLE SHEETS. A SET OF RULES THAT DEFINE THE PRESENTATION OF VISUAL ELEMENTS ON AN HTML PAGE

HTML:

  • HTML --> HyperText Markup Language
  • Hypertext --> TEXT THT CONTAINS LINKS TO OTHER TEXT
  • Markup --> MARK SOMETHING UP TO ANNOTATE, ITS ALL ABOUT CONTENT WRAPPED UP AROUND TAGS
  • HTML --> FOR STRUCTURE
  • HTML COMMUNICATES INSTRUCTIONS TO THE WEB BROWSER. INDICATES HOW THE PAGE SHOULD BE PRESENTED TO THE VIEWER.
  • HTML TAGS ARE CALLED TAG/ELEMENT
  • Browsers ALWAYS interpret HTML sequentially, top to bottom
  • quirks mode --> NON-COMPLIANT PAGES. WHEN THE BROWSER Assumes that the HTML in the web page is NOT following the HTML standard, i.e., not in standards mode. Styles won't work correctly, etc.
  • standard mode --> COMPLIANT PAGES
  • What would happen if you didn't specify the HTML5 declaration? The page will be interpreted in quirks mode
  • BLOCK LEVEL ELEMENTS: (OFFICIALLY NOT PART OF HTML5, BUT STILL USED QUITE OFTEN IN LITERATURE AND REGULAR CODING)
    • RENDER TO BEGIN ON A NEW LINE (BY DEFAULT)
    • MAY CONTAIN INLINE OR OTHER BLOCK-LEVEL ELEMENTS
    • ROUGHLY FLOW CONTENT (HTML5 CATEGORY)
  • INLINE ELEMENTS: (OFFICIALLY NOT PART OF HTML5, BUT STILL USED QUITE OFTEN IN LITERATURE AND REGULAR CODING)
    • RENDER ON THE SAME LINE (BY DEFAULT), BUT CAN BE CHANGED
    • MAY ONLY CONTAIN OTHER INLINE ELEMENTS
    • ROUGHLY PHRASING CONTENT (HTML5 CATEGORY)
